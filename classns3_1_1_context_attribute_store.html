Detailed Description

Stores baseline and runtime context attributes of IoT nodes.

// Zero Trust Context Drift Validator

CLASS: ContextAttributeStore

This class maintains the contextual information of nodes participating in the Zero Trust framework. It stores baseline context values captured during onboarding and continuously updates runtime context values.

The class is responsible for:

  • Maintaining baseline and current context attributes.
  • Computing contextual drift.
  • Updating trust scores based on drift.
  • Providing trust information to policy enforcement modules.

Calculates contextual drift for a node.

Computes the weighted deviation between baseline and current context attributes.

Drift = (wt * timeDeviation)

  • (wc * congestionDeviation)
  • (wp * proximityDeviation)

Higher drift values indicate greater deviation from the trusted baseline context.

Updates trust score using exponential trust decay.

Trust decreases as contextual drift increases. The decay follows an exponential model controlled by the lambda parameter.

Trust = Trust * exp(-lambda * drift)
